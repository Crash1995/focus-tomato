# FocusForge — Задачник и история дней

**Дата:** 2026-04-24
**Статус:** design draft

## Цель

Превратить FocusForge из «помидор + описание сессии» в инструмент, где день описан заранее списком задач, и в конце дня видна метрика «% закрытых задач», а за произвольный период — динамика.

## Рамки

В рамках:
- Список задач на день с ручной сортировкой (drag-and-drop) и чекбоксами.
- Утренний режим планирования (вставка многострочного списка).
- Отдельный full-screen view истории: график за 14 дней + лента по дням + развёрнутая карточка сегодня с опциональным AI-комментарием.
- Миграция модели XP на «XP за закрытую задачу».
- Атомарная запись data.json.

Не в рамках:
- Подзадачи, теги, приоритеты (кроме ручного порядка), дедлайны.
- Синхронизация между устройствами.
- Экспорт истории.
- Новые уровни геймификации (пороги XP не трогаем).

## Поведенческая модель

### Задачи независимы от таймера

Таймер и список задач — две параллельные вещи. Пользователь сам отмечает закрытые задачи в удобный момент (во время работы, в перерыве, не обязательно по окончании помидора).

### Утренний ввод

Когда для текущей даты нет задач, центральная колонка показывает приглашающий экран с `textarea` и кнопкой «Запланировать». Пользователь вставляет многострочный список — парсер превращает каждую непустую строку в задачу:

- Срезаются ведущие маркеры: `-`, `*`, `•`, `1.`, `2.` и т.д., а также чекбоксы `[ ]` / `[x]`.
- Если строка была `[x]` — задача создаётся уже отмеченной (`done: true`), с `completedAt` = момент импорта.
- Строки, ставшие пустыми после срезания, пропускаются.

После появления первой задачи центральная колонка переключается в обычный режим — список + inline-поле «+ Новая задача» внизу. В inline-поле тоже работает построчный парсинг: если пользователь вставит несколько строк через clipboard, добавятся несколько задач.

### Основной UI (3 колонки)

Окно остаётся 900×560, frameless, resizable: false — без изменений.

1. **Левая колонка — Timer.** Без изменений.
2. **Центральная колонка — Задачник.** Табы сверху:
   - **«Задачи N/M»** — открытые задачи дня, сортируемые drag-and-drop. Внизу inline-поле. M — всего задач сегодня, N — закрытых.
   - **«Сделано · K»** — закрытые задачи дня, отсортированные по `completedAt` по убыванию.
   Клик по чекбоксу на открытой задаче → задача с короткой fade-out анимацией (200 мс) уходит во вкладку «Сделано», +25 XP toast. Клик по чекбоксу на закрытой → возвращается в «Задачи», −25 XP.
3. **Правая колонка — Stats.** Lvl, прогресс-бар XP, стрик, «Задач: N/M», «Сессий: N». Плюс кнопка «История».

Старая плашка «Итоги дня» и старый центральный режим AI-отчёта в основном окне **удаляются**. AI-отчёт теперь живёт в History view (см. ниже).

### Модалка завершения помидора

После окончания помидора вместо текущей модалки с полем ввода — мини-модалка с двумя кнопками:
- **«Ещё 🍅»** — стартует новый рабочий помидор.
- **«Break»** — стартует break-таймер (как сейчас).

Звук (уже сделанный 3-секундный колокольный сигнал) играет на открытии модалки.

### History view (full-screen)

Кнопка «История» в stats-колонке открывает full-screen view поверх приложения (абсолютное позиционирование, крышка с кнопкой закрытия). Содержит:

1. **Карточка сегодняшнего дня** (вверху, развёрнута по умолчанию):
   - Дата, метрики (`X/Y задач · Z 🍅 · W XP`).
   - Список закрытых задач дня.
   - Кнопка «AI-комментарий ✨». Опциональна. Вызывает OpenRouter с ключом из settings. Промпт получает: дату, закрытые задачи, число помидоров. В ответ — те же «3 пункта» (главное достижение / паттерн / совет), русский язык.
2. **График за последние 14 дней.** Вертикальные столбцы, высота = % закрытых задач этого дня (`closed / total`). Если в дне 0 задач — столбец высоты 0. Ось X — даты.
3. **Лента дней.** Свёрнутые строки для каждого дня с данными: `дата · X/Y задач · N 🍅 · M XP`. Клик по строке — раскрывается список задач того дня (как у сегодняшней карточки, но без кнопки AI-комментария — AI запускается только для сегодня).

Закрывается крестиком или Esc. При открытии всегда возвращает в состояние «крышка с сегодня наверху».

## Модель данных

### data.json

К существующему объекту добавляется поле `tasks`:

```json
{
  "sessions": [ /* как сейчас */ ],
  "tasks": [
    {
      "id": "t-1714000000000-3",
      "date": "2026-04-24",
      "title": "Веб3: скилл — описание",
      "done": false,
      "order": 0,
      "createdAt": 1714000000000,
      "completedAt": null
    }
  ],
  "stats": { /* как сейчас */ },
  "settings": { /* как сейчас */ }
}
```

Поля задачи:
- `id` — уникальный, формат `t-<createdAt>-<random>`.
- `date` — `YYYY-MM-DD`, дата, к которой привязана задача (день планирования).
- `title` — строка, обрезанная до 200 символов.
- `done` — булев. При переводе в `true` ставится `completedAt = Date.now()`, при возврате в `false` — `completedAt = null`.
- `order` — позиция в списке открытых задач. Целое. После любого drop-а у затронутых задач `order` перенумеровывается 0..n.
- `createdAt` — ms timestamp создания.
- `completedAt` — ms timestamp закрытия (или null).

### Сортировка в UI

- Вкладка «Задачи» (открытые): `done === false`, сортировка по `order ASC`.
- Вкладка «Сделано»: `done === true`, сортировка по `completedAt DESC`.
- Выборка «задачи дня» = `tasks.filter(t => t.date === today)`.

### Стрик

Правило не меняется: стрик продлевается, если `lastActiveDate === yesterday` и вчера было ≥ 4 помидоров. `PRODUCTIVE_DAY_SESSIONS = 4` остаётся.

### XP

- Базовая физика: +25 XP на каждое событие «задача закрыта», −25 XP на событие «задача открыта обратно» (для корректного отката случайного клика).
- Старые `BASE_SESSION_XP = 25` и `DESCRIPTION_BONUS_XP = 10` удаляются из `gamification.js`.
- Уровни остаются те же.
- При импорте через `[x]` в утреннем вводе: считаем, что задача закрывается → +25 XP за каждую отмеченную.

### Миграция существующих данных

Существующие пользователи имеют `tasks: undefined`. В `mergeWithDefaults` добавляется `tasks: []` по умолчанию. Никакой автоматической ретроспективной конвертации старых `sessions[].description` в задачи нет — описания теряют смысл, но сами сессии остаются для стрика и счётчика.

## Технические решения

### Атомарная запись data.json

Текущий `saveData` использует `fs.writeFile` прямо на целевой файл — при сбое питания или crash можно получить повреждённый JSON. Меняется на: `fs.writeFile` во временный файл `data.json.tmp` в той же директории → `fs.rename` на `data.json`. На POSIX rename атомарен, частично записанного состояния не возникает.

### Drag-and-drop

Используем нативные HTML5 drag events (`draggable="true"`, `dragstart`, `dragover`, `dragend`, `drop`) без внешних библиотек. Ожидаемый объём — ≈80 строк в `app.js`. При дропе задача перевставляется в массив и `order` всех задач текущего дня пересчитывается. Сохранение через `saveData()` после дропа.

### AI-отчёт

Функция `sendAIRequest` в `src/main/openRouterClient.js`:
- Принимает `{ closedTasks: Task[], pomodoroCount: number, apiKey }` вместо сессий.
- Системный промпт переписан под новый контент (3 пункта остаются).
- `formatSessions` → `formatTasks`.
- Старая кнопка «Итоги дня» в основном окне удаляется.
- Кнопка «AI-комментарий ✨» в History view показывает loading → текст или ошибку, текст кэшируется на карточке сегодняшнего дня в пределах сессии приложения (не сохраняется в data.json).

### Структура кода

Растёт `src/app.js`. Чтобы не превратить его в 1000-строчный монолит, выделяется два новых модуля:

- `src/renderer/tasks.js` — работа со списком задач: создание, toggle, drag-and-drop, парсер многострочного ввода, фильтрация по дате. Чистые функции + пара функций-рендереров.
- `src/renderer/history.js` — рендер History view: агрегация данных по дням, отрисовка графика и ленты, AI-комментарий.

Оба подключаются в `index.html` через `<script>` до `app.js`. Экспорт — через `window.FocusForgeTasks` / `window.FocusForgeHistory` по тому же паттерну, что уже используется для `FocusForgeCore` в `gamification.js`.

`src/app.js` остаётся точкой входа: связывает DOM с этими модулями и управляет состоянием приложения.

### Удаляемый код

- `sessionModal` и связанные обработчики в `app.js` / `index.html` / `styles.css`.
- `updateSessionXPPreview`, `saveCompletedSession` (в текущем виде) — заменяются на новый хэндлер закрытия помидора.
- `renderSummaryCard` / `renderSummaryLoading` / `renderSummaryError` / `restoreSessionPanel` в `app.js` — уходят в `history.js` в переосмысленном виде.
- Ветка UI «рендер списка сессий» в центральной колонке (`renderSessions` в текущем виде) — удаляется.

## Ошибки и граничные случаи

- **Пустой день в истории.** Если в дне нет задач и сессий — строка в ленте всё равно показывается с `0/0 задач · 0 🍅`, но столбец графика пустой (высоты 0).
- **Задача без title после парсинга.** Пропускаем при импорте, ничего не создаём.
- **Вставка 200 строк.** Технически работает, но чтобы не дать создать мусор: лимит на импорт — 50 задач за одну операцию. Сверху — уведомление «Добавлено 50 из N строк, остальные отброшены».
- **Перевод даты в полночь.** Запущен `setInterval` раз в 30 секунд, сравнивающий текущее значение `getToday()` с закэшированным. При изменении — центральная колонка перерисовывается от нуля (обычно — в режим утреннего приглашения, если для новой даты нет задач). Задачи вчерашнего дня остаются в `tasks[]` (для истории), но в UI «Сегодня» они не видны. Вчерашние открытые задачи не переносятся автоматически — пользователь осознанно планирует новый день.
- **AI-комментарий без ключа.** Кнопка «AI-комментарий ✨» всегда видна, но при клике без ключа открывается существующая модалка с полем ввода ключа (как сейчас).
- **AI-комментарий только для сегодня.** Для исторических дней кнопка не показывается — иначе раздуем счётчик платных запросов при просмотре истории.
- **Откат закрытия задачи (−25 XP).** Если totalXP уже меньше 25, не уходим в минус — ставим totalXP = 0. Уровень пересчитывается от totalXP.

## Критерии приёмки

1. При первом запуске пустого дня центральная колонка показывает textarea с placeholder. Вставленный многострочный список превращается в задачи; `- [ ]` / `- [x]` / `* ` / `1. ` корректно парсятся.
2. После появления первой задачи центральная колонка — список задач с табами «Задачи N/M», «Сделано · K» и inline-полем ввода внизу.
3. Drag задачи в списке «Задачи» сохраняет порядок после перезапуска приложения.
4. Клик по чекбоксу открытой задачи: XP растёт на 25, задача уезжает во вкладку «Сделано». Клик по ней там же — XP падает на 25, задача возвращается.
5. Модалка «Ещё 🍅 / Break» появляется после помидора, текстового поля нет.
6. Кнопка «История» в stats-колонке открывает full-screen view. Сегодняшний день — развёрнутая карточка сверху, график за 14 дней (столбцы), лента прошлых дней.
7. Кнопка «AI-комментарий ✨» на карточке сегодняшнего дня вызывает OpenRouter с названиями закрытых задач, отрисовывает ответ.
8. Стрик продолжает считаться по помидорам.
9. Если во время записи data.json приложение крашнется, файл не повреждается (атомарная запись).
10. Существующие пользователи с `data.json` без `tasks` получают `tasks: []` при первой загрузке.
